Breeze .NET server extensions for enforcing entity authorization and other such business rules in a pipeline

It's time for business. Breeze business

Let's face it. Breeze Js is awesome and it increases your productivity when building data-centric web apps. It works especially well when you opt in to its server-side components designed to work with ASP.NET Web API.

In our experience, we've found that it would be beneficial to add to this awesomeness with more explicit control over what happens on the server when trying to do the CUD of CRUD.

Features

  • Authorization rule validation data-annotations style or fluent configuration via a registry
  • Pipeline processing for injecting any set of business rules

Installation

Breeze.BusinessTime is hosted on NuGet. Search for it in the Package Manager, or simply run the following command in the Package Manager Console in Visual Studio.

PM> Install-Package Breeze.BusinessTime

Usage

  1. Use with your normal Breeze Web API controllers
  2. Replace your current context provider with AuthorizedDbContextProvider
  3. Decorate your model classes (if code first) with AuthorizeEntity attributes or use the fluent registry API if you'd like to see all of your authorization rules in once place.
  4. Bob's your uncle
